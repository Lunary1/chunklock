## ü§ù Contributing

### When to archive a document:

1. **Milestone completion** - Document marks a specific project completion
2. **Historical significance** - Contains important decision-making context
3. **No longer active** - Information superseded by newer documentation
4. **Reference value** - May be useful for understanding past choices

### When NOT to archive:

1. **Active user guides** - Belongs in `user-guides/`
2. **Current developer docs** - Belongs in `developer/`
3. **Wiki content** - Belongs in `wiki/`
4. **Truly obsolete** - Just delete if no historical value
